# Documentation

This directory contains project documentation.

- `architecture.md`: implemented architecture and service boundaries.
- `workflow.md`: implemented lifecycle and workflow rules.
- `schema.md`: persisted model and enum reference.
- `api-spec.md`: implemented API surface and behavior.
- `status.md`: current capabilities, completed milestones, and test status.
- `development-plan.md`: current state, target state, and remaining work for the next implementation phase.
- `use-cases.md`: intended writing-agent use cases beyond the current academic-paper prototype.
- `limitations.md`: current product and engineering limitations.
- `roadmap.md`: recommended next implementation directions.
- `handoff.md`: quick-start notes for a future Codex or human developer.
- `implementation-brief.md`: original Codex-ready implementation brief.
